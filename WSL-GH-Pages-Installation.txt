| This procedure gets around the symbolic link issues with WSL and git by
| mounting a windows directory containing the repo on WSL

Installation
============

Install the Ubuntu App from the Microsoft Store

Launch Ubuntu and set the username and password

Then in the WSL Window: -
#
sudo apt-get update -y && sudo apt-get upgrade -y
#
sudo apt-get install ruby ruby-dev build-essential 
#
sudo apt-get inatall zlib1g-dev
#
sudo apt-get install libcurl4-openssl-dev
#
sudo gem update
#
sudo gem install jekyll bundler
#

Back in Windows create a folder for the repo eg: -

C:\Users\<username>\tvrename

Back in the WSL window

#
cd ~
#
mkdir tvrename
#
sudo mount --bind /mnt/c/Users/<username>/tvrename tvrename
#
# Contents of the Windows directory is now available in WSL
#
git clone https://www.github.com/andy-abtec/tvrename.git tvrename
#
# GitHub repo cloned into the local tvrename directory
#
cd tvrenane
#
vi Gemfile
	source 'https://rubygems.org'
	gem 'github-pages', group: :jekyll_plugins
#
# To update...
#
sudo apt-get update -y && sudo apt-get upgrade -y
#
# Then...
#
bundle update
#
# To launch the local web server...
#
bundle exec jekyll serve
#

Shrink this window down but leave it somewhere the last few lines can
be monitored. Errors with the Jekyll server will be displayed here.

And browse to http://127.0.0.1:4000 on Windows

Fire up a second WSL session and cd th tvrename

The git commands can be run here

#
# To stop the prompting when running git push etc.
#
git config --global user.email "<Email registered with Repo>"
git config --global user.name "<Repo username>"
#

Files can be edited copied etc in the Windows environment (C:\Users\<username>\tvrename)
